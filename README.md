# GETMANENKO DESSERT — Премиальные торты и десерты

Современный, адаптивный Landing Page сайт-портфолио для кондитера в г. Тимашевск. Сайт разработан с использованием премиального минималистичного дизайна, поддерживает переключение тем оформления (Lux Dark Mode), 3D Tilt-эффекты и обладает встроенной бессерверной Cloud CMS.

## 🛠 Технический Стек

*   **Frontend**: HTML5, Vanilla JavaScript (ES6+), Tailwind CSS (через CDN).
*   **Шрифты и Иконки**: Inter (для отличной поддержки кириллицы), EB Garamond, Material Symbols Outlined.
*   **Интеграция с Облаком**: Firebase SDK (Firestore для хранения текстов/структуры и Cloud Storage для хранения изображений).
*   **Хостинг**: Cloudflare Pages (высокая скорость загрузки, автоматический сборщик).

---

## 🚀 Основной Функционал

1.  **Каталог начинок**: Отображает 25 изысканных начинок тортов. Доступен текстовый поиск по ингредиентам, фильтрация по категориям (Муссовые, Шоколадные, Ягодные, Классические) и подробные карточки с составом.
2.  **Витрина десертов**: Презентация флагманских десертов (например, Сырный пирог с грушей) с интерактивным переключением ракурсов («Целиком» и «В разрезе»).
3.  **Галерея работ**: 27 реальных фотографий тортов кондитера, разбитых по категориям. Включает полноэкранный Лайтбокс (просмотрщик) с поддержкой жестов смахивания на телефонах и горячих клавиш на ПК. Названия тортов скрыты — отображаются только теги категорий.
4.  **Lux Dark Mode**: Плавная смена темной и светлой темы с сохранением выбора в `localStorage` и анимацией вращения иконки переключателя.
5.  **3D Tilt Эффект**: Карточки начинок и слайды галереи наклоняются вслед за курсором мыши, создавая ощущение глубины, с динамическим позиционированием светового блика (`.tilt-shine`).
6.  **Видео-превью**: Промо-видео процесса приготовления открывается в кастомном модальном окне с автовоспроизведением и сбросом iframe при закрытии (для остановки звука).

## 🔐 Панель Администратора (CMS)

Нажмите на иконку «Замочек» в верхнем правом углу (или перейдите по ссылке «Админ-панель» в подвале).
*   **Пароль по умолчанию**: `admin`

После входа на сайте активируется **режим редактирования**:
*   На карточках начинок, десертов и галереи появляются кнопки **Редактировать** (карандаш) и **Удалить** (корзина).
*   В начале каждого блока появляется пунктирная карточка **«Добавить»**.
*   Модальные окна администратора поддерживают выбор шаблонов фото из уже существующих, а также **загрузку файлов напрямую с устройства** (телефона или компьютера).

## ☁️ Настройка Облачного Режима (Firebase Cloud CMS)

Сайт работает в двух режимах хранения изменений:
1.  **Автономный режим (по умолчанию)**: Картинки конвертируются в Base64, данные сохраняются в `localStorage` вашего браузера. Изменения видны только вам локально.
2.  **Облачный режим (для заказчика)**: Все изменения сохраняются в базу данных Firestore и хранилище Firebase Storage. Изменения видны всем пользователям интернета.

### Инструкция по подключению облака:
1.  Создайте проект в [Firebase Console](https://console.firebase.google.com/) (это бесплатно).
2.  Включите **Cloud Firestore** в тестовом режиме.
3.  Включите **Firebase Storage** в тестовом режиме.
4.  В начале тега `<script>` в файле `index.html` найдите объект `firebaseConfig` и вставьте ваши ключи из настроек Firebase:
    ```javascript
    const firebaseConfig = {
        apiKey: "ВАШ_API_KEY",
        authDomain: "PROJECT_ID.firebaseapp.com",
        projectId: "PROJECT_ID",
        storageBucket: "PROJECT_ID.appspot.com",
        messagingSenderId: "SENDER_ID",
        appId: "APP_ID"
    };
    ```
5.  Залейте обновленный `index.html` на Cloudflare Pages.

## 🎨 Выполненный Рефакторинг и Оптимизация

В ходе финального этапа разработки были проведены следующие улучшения:
*   **Централизация модальных окон**: Логика плавных переходов открытия/закрытия всех 6 типов модальных окон и клика по их фону вынесена в единый объект `ModalManager`.
*   **Удаление избыточных файлов**: Полностью удалена серверная часть `server.js` (перезапись файлов на сервере больше не требуется, так как сайт переведен на Cloud CMS).
*   **Сжатие медиа-ресурсов**: Все 60 стоковых изображений в папках `tort/` и `fillings/` были сжаты с помощью Pillow без потери визуального качества. Суммарный размер загрузки сокращен с **6.7 МБ до 4.4 МБ (на 35%)**, что решает проблемы с тайм-аутами при загрузке на хостинг.
*   **Оптимизация Git и Surge**: Добавлены `.gitignore` и `.surgeignore` для исключения системных файлов и логов из деплоев.
